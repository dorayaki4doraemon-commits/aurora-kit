## 0. TL;DR(30秒で把握する)

- **1ファイル完結の静的HTML** (`atmos.html`)。ビルドステップなし。外部依存はGoogle Fonts CDNのみ。
- 2つの見た目モードを1つのDOMで共有する: **Aurora(夜)** と **Rainbow(昼)**。切り替えは `<html data-theme="day">` の有無のみ。CSSカスタムプロパティが全ての差分を吸収する。
- 核となる思想は「**自家発光は、太陽ただひとつ**」。CTA以外は全て反射(夜)か分光(昼)で光る。この規範はコードにも表れている(§7参照)ので、勝手に「光る要素」を増やさないこと。
- パフォーマンス上の既知の急所は `backdrop-filter` + SVGフィルターの組み合わせ(§9)。これは意図的に1パネルだけに制限してある。範囲を広げるときは必ず実機計測してから。
- **触ってよい**: 色トークンの値、新しいセクションパネルの追加、環境生物の追加、コピーの調整。
- **触る前に一度立ち止まるべき**: `--breath`(世界の心拍)の値、Glass層の`backdrop-filter`適用範囲、`data-theme`の切替方式そのもの。

## 2. コンセプト(実装判断の根拠)

新しい機能を足す・既存を直す際は、まずこのセクションに立ち返って「思想に反していないか」を確認すること。デザインの一貫性はコードレビューの基準でもある。

### 2.1 命名の由来
**ATMOS** = Atmospheric Optics(大気光学)。オーロラ・虹・月暈・流れ星は全て大気光学現象であり、本システムは「空の光の物理をUIに翻訳するシステム」である。

### 2.2 2つのモード

| | Aurora Mode(夜) | Rainbow Mode(昼) |
|---|---|---|
| 設定 | 一年でいちばんオーロラが濃い夜 | 雨上がりの、いちばん光が澄んだ昼 |
| 光の届き方 | 反射(月が太陽光を反射) | 分光(水滴が太陽光を7色に割る) |
| 家計簿の通貨 | 光子(ph/s) | 彩度 |
| 光源の描写 | **描いてよい**(月は直視できる) | **絶対に描かない**(太陽は直視できない) |

### 2.3 不変条件(MUST — 変更する場合は要議論)

1. **自家発光はCTA(`.cta-firefly`)ただひとつ**。新しい常時発光アニメーションを他要素に追加しない。
2. **Rainbow Modeで太陽そのものを描画しない**。太陽高度・色温度・影の向きという"証拠"のみで存在を示す(`updateSun()` 参照)。
3. **常灯 / 翳りは異常のみ**。エラー以外の要素を常時ハイライトさせない。
4. **世界の心拍は `--breath: 4s` の倍数**。新しい環境アニメーションの周期は 4 / 8 / 12 / 16 / 20 / 24s のいずれかにする(§7.3)。
5. **`data-theme="day"` 以外に状態を増やさない**。テーマは常に二値(夜/昼)。三値化する場合は本仕様書とトークン設計を先に見直す。

## 3. デザイントークン

`:root` に定義。すべてのコンポーネントはトークン参照のみで実装し、ハードコード値を書かない(v0.2までにハードコードの残存箇所を洗い出すこと。既知の残存箇所は§10参照)。

### 3.1 Spacing / Radius / Motion

```css
--space-1:4px;  --space-2:8px;  --space-3:12px; --space-4:16px;
--space-5:24px; --space-6:32px; --space-7:48px; --space-8:64px;

--radius-sm:10px;  --radius-md:17px;  --radius-lg:26px;
--radius-pane:30px 30px 22px 22px;      /* 上下非対称。水に浮くもの=上が緩み下が締まる */
--radius-pane-in:26px 26px 18px 18px;

--duration-fast:.15s;  --duration-med:.4s;  --duration-slow:1.2s;
--breath:4s;                              /* 世界の心拍。§7.3必読 */
--ease-out:cubic-bezier(.2,.6,.3,1);
--ease-spring:cubic-bezier(.34,1.6,.64,1);
--ease-contract:cubic-bezier(.55,0,.2,1); /* 速く収縮・遅く弛緩(クラゲ/錦鯉) */
```

これが本システムの核。**同じトークン名**に、モードで異なる値が入る。新しい色を足すときも「役割」から命名し、直接の16進値をコンポーネントCSSに書かない。

| Token | 役割 | Aurora値 / 名称 | Rainbow値 / 名称 |
|---|---|---|---|
| `--teal` | 主役 | `#19E3B1` Aurora Teal | `#086B50` Spectrum Green |
| `--cyan` | 副役 | `#35C4F0` Aurora Cyan | `#0A5CA8` Spectrum Blue |
| `--violet` | 特別 | `#7A6CF0` Aurora Violet | `#5A35A8` Spectrum Violet |

## 4. テーマシステムの実装詳細

### 4.1 切替の仕組み
```js
document.documentElement.setAttribute('data-theme', 'day'); // Rainbow Mode
document.documentElement.removeAttribute('data-theme');      // Aurora Mode(デフォルト)
```
CSS側は `[data-theme="day"] { --teal: #086B50; ... }` で該当トークンを上書きする形。**JSでインラインスタイルを書き換える方式は使わない**(トークンの一元管理が崩れるため)。

### 4.2 テーマ切替トグルの実装(`#themeToggle`)
円形ボタン内にSVGの「天体運行トグル」を内包。月と太陽が同一DOM内に共存し、`[data-theme="day"] .tgl-orbit { transform: rotate(180deg); }` で公転して入れ替わる。星は瞬き、雲は流れる。**絵文字は使用しない**(v0.1以前に絵文字ベースの実装があったが、角の設計・反射思想のどちらにも従わないため廃止した。再度絵文字に戻すことは設計原則への逆行になる)。

クリックハンドラ: `#themeToggle` の `click` イベント → `data-theme` トグル + `aria-pressed`/`aria-label` 更新 + `updateSun()` 再計算 + 虹の抽選タイマー起動(`tryRainbow`)。

### 4.3 モード専用の生物・環境要素

夜専用要素(`display:none` in `[data-theme="day"]`): `#auroraLayer`, `#moonLayer`, `#starLayer`, `#owl`, `.jellywrap`

昼専用要素(デフォルト非表示、`[data-theme="day"]`で表示): `#daySky`, `#tempLayer`, `#rainlayer`, `#rainbow`, `.koiwrap`, `#swallow`

**姿を変えるが消えない要素**(実装上の重要な区別 — 削除ではなくCSS上書きで対応):
- 蛍(`.ffly`)→ 泡: 同一JS物理エンジン(`flyLoop`)のまま、CSSで見た目だけ泡に変える
- 湖(`.lake`)→ 水たまり: 同一DOM・同一クリック波紋ロジック、配色のみ変更

この区別は重要: 新しい環境演出を作るとき、「昼夜で完全に別物」なのか「同じ仕組みで姿だけ変わる」のかを最初に決めること。後者の方が実装・保守コストが低く、システムの一貫性(「同じ風景の違う時刻」という核概念)にも合致する。

## 5. コンポーネント・カタログ

### 5.1 ボタン
| クラス | 形状 | 状態 |
|---|---|---|
| `.cta-firefly` | 尾を持つ雫(左下角のみ9px) | default / hover(scale1.045) / active(scale.98) / **disabled**(発光停止+彩度40%) |
| `.ice-btn` | 六角ファセット(`clip-path`) | default / hover(縁強調) / active |
| `.halo-btn` | pill+外周リング(サイズ固定)。`--halo-op`で3段階の明瞭さ(.2/.55/1) | hoverで暈がやや明瞭化 |
| `.reflect-btn` | 欠けた氷(一角のみ5px) | hover/タップ: sheen+frost+波紋(`.ripple-host`) |

**disabled規約**: `opacity:.38` + アニメーション全停止 + `cursor:not-allowed` + `filter:saturate(.4)`。「光る資格の一時停止」の表現として統一する。

### 5.2 入力・トグル
- 入力: 通常時はフォーカスで銀の反射(`--jelly`色のアウトライン)。**error状態**は `.inp.error` + `aria-invalid="true"` + `aria-describedby` で `.err-msg` と紐付け必須。Rainbow Modeではエラーは赤ではなく`grayscale`の「翳り」。
- トグル: OFFは月光銀ノブ、ONはオーロラ緑ノブ+外周グロー。移動イージングは `--ease-spring`。

### 5.3 タッチ・パリティ(hoverが存在しない端末への対応)

| hover演出 | タッチでの等価実装 |
|---|---|
| sheen+frost(反射) | `pointerdown` で `.tap-fx` クラス付与 → 同一アニメ再生 |
| 波紋 | 同時発火 |
| 押下感 | 全ボタン `:active { transform: scale(.965) }` |
| 見出しの文字間の呼吸 | `@media (hover:none)` では常時開いた状態に固定 |

新しいインタラクティブ要素を追加するときは、**必ずタッチ版の代替を同時に実装すること**。hover専用のまま出荷しない。

## 6. 環境レイヤー・生物の挙動仕様

| 要素 | 挙動 | 実装関数/仕組み | 頻度 |
|---|---|---|---|
| 月 | 実月齢を計算し正しい月相を描画。輝面比がグロー強度に連動。スクロールでアーチ運行 | `computeMoonPhase()`, `moonLitPath()`, `onScroll()` | 常時 |
| 月クラゲ×2 | 解剖精度(裂片8・放射管・生殖腺・口腕・触手位相ずれ)。カーソル接近で傾く | `.jelly-tilt`, `tiltLoop()` | 常時(2体目8s周期) |
| 蛍/泡×6 | カーソルに緩く惹かれ、近すぎると逃げる。速度上限0.55px/f | `flyLoop()` | 常時 |
| 梟 | 稀に現れ緑の瞳で瞬く | `owlVisit()` | 35〜65s間隔、8s滞在 |
| 燕 | 稀に横切る(梟の昼版) | `swallowVisit()` | 30〜60s間隔 |
| 流れ星 | セッション中1度だけ | 固定タイマー(`setTimeout`) | 9〜29s後 |
| サブストーム | オーロラが4秒だけ激しく踊る | `substorm()` | 40〜90s間隔 |
| 通り雨→虹 | 雨脚が数秒戻り、その後に一日一度の虹 | `shower()`, `tryRainbow()` | 50〜100s間隔 |
| 太陽高度 | 実時刻から高度・色温度・影の向きを算出 | `updateSun()` | 60s毎に再計算 |
| 湖/水たまり | 空の反射。クリックで波紋×3 | `lakeEl.addEventListener('click', ...)` | 常時 |

新しい生物・現象を追加する際のチェックリスト:
1. 夜専用か昼専用か、それとも§4.3の「姿を変える」パターンか決める
2. 出現頻度は「常時」「稀(数十秒間隔)」「一度きり(セッション内1回)」のどれかに分類する
3. 周期がある場合は`--breath`の倍数にする
4. `prefers-reduced-motion: reduce` 時の代替(即時表示 or 非表示)を必ず用意する
5. `aria-hidden="true"` を付与する(装飾要素であるため)

## 7. 光の文法(インタラクションの意味体系)

新しいフィードバック(通知・ローディング・完了サインなど)を実装する際は、この4動詞のどれかにマッピングして実装すること。独自の新しい光り方を発明しない。

| 動詞 | 意味 | 夜の実装 | 昼の実装 |
|---|---|---|---|
| 明滅/きらめき | 呼びかけ。周波数=緊急度 | 通知ドット(`blink-slow`/`blink-fast`) | 水面のグリント |
| 脈動/揺らぎ | 進行中 | クラゲの呼吸ローダー | 水中のコースティクス |
| 波紋/虹のアーチ | 完了。一度だけ広がり消える | `.ripple-host.fired` | `#rainbow.arch` |
| 反射/水滴が結ぶ | 接触の応答 | hover銀スペキュラ(sheen) | 分光フリンジ(赤/紫の縁) |
| 常灯/翳り | **異常のみ** | `.steady`(赤) | `grayscale`+影 |

## 8. Glass層(ガラス光学)の実装

### 8.1 基本層(全パネル・全ブラウザ対応)
1. 非対称スクワークル角+内側4pxの第二の縁(四辺で色が異なる: 上=白/下=シアン/左右=色収差)
2. 縁の色収差(左シアン1.5px/右バイオレット1.5px)
3. 上端の鋭いスペキュラライン+楕円ハイライト(`nth-of-type`でパネル毎に位置差)
4. 下端の内部反射
5. 背後の色の滲み+青緑の色つき影

### 8.2 屈折層(**1パネルのみ**・Chrome系ブラウザ)
```
feTurbulence(fractalNoise, 1 octave) → feDisplacementMap(scale:14)   // 屈折
feGaussianBlur(SourceAlpha) → feSpecularLighting + fePointLight       // 動的鏡面反射
```
- 光源座標(`fePointLight`)は**このページの月の実座標にJSで連動**(`onScroll()`内)。primitiveUnitsは`objectBoundingBox`でパネルサイズに依らず共有。
- `@supports not (backdrop-filter: url(#x))` で非対応ブラウザは自動的に基本層のみにフォールバック。

**重要な制約(v0.1時点で確定済み)**: 屈折層は6パネル全部に適用した結果、体感できるレベルで重くなることを実機検証済み。**1パネルより広げる場合は必ず実機のフレームレート計測を行うこと**。安易な拡張を禁止する。

### 8.3 パフォーマンス規約

| 項目 | 規約 | 理由 |
|---|---|---|
| 屈折フィルター適用範囲 | 1パネルまで | §8.2の実測結果 |
| DOM書き込み頻度 | rAF毎の書き込み禁止。500ms間隔にバッチ | 光子の家計簿で実際に発生した問題を修正済み |
| 強制レイアウト(`getBoundingClientRect`) | 220ms間隔+マウス静止時スキップ | クラゲ傾き判定の最適化済み |
| blur半径 | 上限32px、`will-change:transform`併用 | オーロラのGPU合成コスト対策 |
| 画面外要素 | `content-visibility:auto` + `contain-intrinsic-size` | パネル全体に適用済み |
| Calm/タブ非表示時 | 屈折層・flyRAFを`display:none`/停止 | 描画コストごと除去(演出停止だけでは不十分) |

## 9. アクセシビリティ

- 文書構造: `h1` → `h2`(セクション、実タグ化済み)→ `<main>` ランドマーク、`<nav aria-label>`(ドットナビ)
- スキップリンク実装済み(`.skip-link`、フォーカス時のみ表示)
- コントラスト: §3.3の検証値。AA未満の組み合わせを新規追加しない
- `prefers-reduced-motion: reduce` で全アニメーション停止+出現演出は即時表示に切替
- フォーカスリングは全コンポーネントで銀(`--jelly`/`--moonsilver`系)に統一(「反射」の思想を貫く)
- error状態は色だけに依存しない(`aria-invalid` + テキストメッセージ併用)
- 装飾要素(空・生物・湖)は全て `aria-hidden="true"`

## 付録A: コントラスト計算式(WCAG 2.1)

```python
def luminance(hex_color):
    r, g, b = [int(hex_color[i:i+2], 16)/255 for i in (0, 2, 4)]
    def f(c): return c/12.92 if c <= 0.03928 else ((c+0.055)/1.055)**2.4
    return 0.2126*f(r) + 0.7152*f(g) + 0.0722*f(b)

def contrast(fg, bg):
    l1, l2 = luminance(fg), luminance(bg)
    if l1 < l2: l1, l2 = l2, l1
    return (l1 + 0.05) / (l2 + 0.05)
```
新しいテキスト色を追加した場合、この関数で計算し§3.3の表に追記すること。
